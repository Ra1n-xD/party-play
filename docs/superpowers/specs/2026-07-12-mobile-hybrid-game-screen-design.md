# Мобильный гибридный дизайн GameScreen

Дата: 2026-07-12

Статус: утверждено пользователем

## Цель

Объединить мобильную информационную архитектуру ветки `feature/new-design` с визуальным языком и компактными управляющими элементами `feature/new-design-codex`. Изменяется только основной игровой экран. Игровая логика, сервер, сокет-события, роли и фазовая маршрутизация остаются без изменений.

## Сохраняем из feature/new-design

- три мобильные вкладки `Игроки`, `Персонаж`, `Ситуация`;
- компактный список участников, показывающий только публично раскрытые данные;
- отдельное крупное и читаемое личное досье;
- отдельное представление катастрофы, бункера и угрозы;
- существующие доступные модальные окна и диалог управления хоста;
- текущую компонентную структуру `GameScreen`.

## Переносим из feature/new-design-codex

- тёмную зелёно-графитовую палитру с золотым основным акцентом;
- верхнюю шапку игры с брендом, кодом комнаты, состоянием соединения, управлением хоста и выходом;
- компактную мобильную панель игровых действий в одну строку;
- сенсорные области не меньше 44×44 px;
- спокойные тонкие границы, зелёные состояния подключения и золотое выделение текущего хода.

## Компоненты

### GameRoomHeader

Новый презентационный компонент получает только существующие данные и callback-функции:

- `roomCode: string | null`;
- `connected: boolean`;
- `canManageGame: boolean`;
- `onOpenHostControls: () => void`;
- `onLeaveRoom: () => void`.

Он не обращается к сокету и не изменяет состояние самостоятельно. Управление показывается только хосту. Выход вызывает существующий `leaveRoom`.

### GameStatusHeader

Продолжает показывать фазу, инструкцию, таймер, прогресс раундов и пропуск обсуждения. Кнопка `Управление` удаляется из этого блока, потому что переезжает в `GameRoomHeader`. На мобильном статусный блок остаётся компактным и не повторяет данные шапки комнаты.

### MobileGameTabs

Логика и семантика вкладок не меняются. Внешний вид переводится на графитовые поверхности, зелёные границы и золотой активный акцент.

### Панель действий

На ширине до 768 px доступные действия располагаются в одну строку. Если присутствуют две кнопки, вторичное действие занимает меньшую долю, главное раскрытие характеристики — большую. Текст может переноситься максимум на две строки. Высота панели ориентировочно 64–72 px вместе с отступами и safe-area.

На ширине до 360 px панель сохраняет одну строку и не вызывает горизонтальный скролл. Все кнопки имеют высоту не меньше 44 px.

## Визуальные токены

- фон: `#07100c`;
- основная поверхность: `#101713`;
- приподнятая поверхность: `#141b17`;
- основной текст: `#f3f1e8`;
- вторичный текст: `#8e9991`;
- граница: `rgba(178, 199, 185, 0.14)`;
- золотой акцент: `#efbd58`;
- зелёный статус: `#76c69e`;
- опасное действие: `#d9786c`.

Цвета типов характеристик сохраняются, поскольку помогают быстро различать данные.

## Адаптивность

### До 768 px

- шапка комнаты располагается над статусом фазы;
- второстепенные текстовые подписи состояния соединения скрываются, но иконка и доступное имя остаются;
- вкладки сохраняют три равные сенсорные области;
- игроки остаются компактным одноколоночным списком;
- нижняя панель занимает одну строку;
- контент получает нижний отступ с учётом фактической панели и `safe-area-inset-bottom`.

### От 769 px

- сохраняется существующая desktop-композиция доски игроков и личного досье;
- шапка комнаты остаётся полноширинной внутри максимальной ширины GameScreen;
- панель действий может использовать две равные колонки.

## Доступность

- кнопки шапки имеют доступные имена;
- управление скрыто для не-хоста;
- мобильные вкладки сохраняют `tablist`, `tab`, `tabpanel` и клавиатурную навигацию;
- focus-стили переводятся на золотой акцент;
- основные сенсорные области не меньше 44 px;
- существующие focus trap, Escape и восстановление фокуса модальных окон сохраняются.

## Проверка

- `npm run test:game-screen`;
- `npm run build`;
- точечная проверка Prettier изменённых файлов;
- браузерная проверка с восемью игроками на 390×844 и 320×800;
- отсутствие горизонтального скролла;
- высота мобильной панели действий не больше 72 px без safe-area;
- проверка вкладок, управления хоста и выхода из комнаты;
- проверка одной и двух доступных кнопок в нижней панели.

## Критерии приёмки

1. Мобильные вкладки и компактный список игроков сохранены.
2. Экран использует зелёно-золотую стилистику Codex-макета.
3. В шапке доступны код комнаты, соединение, управление хоста и выход.
4. Мобильная панель действий находится в одной строке и имеет высоту не больше 72 px.
5. На 320 px нет горизонтального скролла, а основные кнопки не меньше 44 px.
6. Другие игровые экраны, сервер и Socket.IO-контракты не изменены.
